1 Overview

A4 delivered gates 1–4 as pure predicates (src/lsb/signals/gates.py) and a short-circuiting evaluate() engine. A5 completes the M7 entry qualifier: gates 5–8 plus the eight-gate conjunction — a setup qualifies only when all eight pass on the same evaluation.

The authoritative gate list is Blueprint v2.1 §6.1 "The Eight Entry Gates (M7)" (LSB_Master_Blueprint_v2.1.pdf), with detail rules from Requirements v2.0 §8 (entry checklist) and §9 (structural stop / R:R / target hierarchy). Everything stays a pure function over a candle window: no broker, no clock, no side effects — per the A5 prompt and the Blueprint's "deterministic core" rule.

Discipline carries over from A4 verbatim: each gate is test-driven against hand-labeled synthetic scenarios — 1 pass + ≥2 near-misses per gate, labeled from the spec before the predicate is coded — and a new shuffle property test: shuffled candle history must never qualify at a rate comparable to real history.

2 The eight gates & the pure-function boundary

Blueprint v2.1 §6.1, verbatim. Gates 1–4 are done. The key A5 finding: gates 7 and 8 each contain a runtime sliver (news feed, live drawdown/tier state) that cannot exist in a pure Phase-A engine. The Blueprint itself defers M11/M12 live wiring to Week 5 / Phase B. So A5 implements the deterministic, candle-derivable subset of each gate and represents the runtime sliver as an explicit, documented always-pass stub — never a silent omission.

R:R needs an entry price, but entry is a stop-order trigger (§10.1) not the close — circularity could distort the ratio.Use the spec's structural definition: entry = rejection-candle trigger per §10.1, stop = §9.1, target = §9.4. risk.py takes them explicitly; no broker fill assumed. Pinned to worked examples in test_risk.py.
Float accumulation in R:R / target math → nondeterministic hashes (breaks the Gate GA-1 determinism contract at A8).Pure functions, fixed iteration order, no dict-order sums; determinism unit-asserted now (A4 doctrine) so A8's CI gate inherits a clean base.
Shuffle test is flaky — a threshold on "comparable rate" that passes/fails by luck.Seed the shuffle deterministically; assert a wide margin (real ≫ shuffled, e.g. shuffled rate ≤ ¼ real or near-zero) rather than a tight numeric equality. Document the seed.
BTCUSD ~1y history is short — fewer qualifying windows for the property test.Property test runs per-instrument on whatever window exists (gates are pure); short history is fine for a rate comparison. The walk-forward bite stays an A9 concern, already flagged in PHASE_STATUS.
Scope creep — temptation to pull in full M8 position sizing or an M11 state machine.Hard stop: risk.py is stop + R:R only (no equity, no lots). Sizing, drawdown state, news feed, and the operating machine are explicitly Phase B. The conjunction + shuffle test is the A5 finish line.
