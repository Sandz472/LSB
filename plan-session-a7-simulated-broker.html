Session A7 — SimulatedBroker: pessimistic fill model

Implementation plan · 2026-06-19 · LSB Strategy Bot (Phase A · Week 2)

1 Overview

A6 shipped the event-driven backtest core driven by NaiveBroker — an optimistic fill model that fills any touched trigger at the exact trigger price. A7 replaces it with SimulatedBroker, implementing the Blueprint v2.1 §7.1 pessimistic fill model in full: spread cost, ATR-state-scaled slippage (EXTREME doubles it), gap-through-stop fills at the gap open, and commission/swap. The whole point of §7.1 is honesty — “a thin edge that costs erase is not an edge.”

The fill seam is already designed for this swap: run_backtest(..., broker=...) injects any Broker Protocol implementation, and BrokerCosts already exists in the config schema with all five cost fields populated per instrument. The work is concentrated in broker.py plus one honesty fix to the stop-exit path in loop.py, then worked-example unit tests.

Acceptance (Impl Plan v3.0 A7): a sample month replays end-to-end producing signal rows for 100% of candles; every fill in the test set matches its hand-computed value; no fill is ever better than the pessimistic model allows.

D3 How are commission & swap modelled when equity/P&L is A10's scope? (c) fails the A7 prompt (“commission/swap … in full”). (b) — compute and record the cost figures at fill/close (honest, deterministic, testable) without folding them into R. A10 consumes them for equity. Commission stored as currency/lot; swap as price-units (nights × swap_pts × pip) — units documented on the fields so A10 knows the conversion.
D4 Spread application: entry-only or round-trip? (a) full spread at entry · (b) half at entry + half at exit · (c) entry-only degrade, exit at mid Round-trip is most honest but needs spread modelled on both legs. The §7.1 text ties spread to fills generally. (a) full modelled spread degrades the entry fill; exits already pay slippage+gap. Entry-only-full and half-entry/half-exit cost the same round-trip total — (a) is simpler with no less pessimism.
D5 Slippage sign / direction — Pessimistic = always against the position. Long entry fills higher, short entry fills lower; stop exits fill worse than the stop. Slippage is always added in the hurting direction.

4 The cost model (precise)

All quantities in price units. pip = config.pip_size; spread_pts, slip_mult, etc. from BrokerCosts.

spread_price = (candle.spread  if candle.spread is not None  else spread_pts) * pip
slip_atr     = slippage_atr_mult * atr_value_at_fill            # ATR already in price units
slip_price   = slip_atr * (2.0 if atr_state == "EXTREME" else 1.0)

# ---- ENTRY FILL (Buy Stop / Sell Stop) ----
LONG :  base = max(trigger, candle.open)        # gap-through: open above trigger fills at open
        fill = base + slip_price + spread_price
        # require candle.high >= trigger to fill at all
SHORT:  base = min(trigger, candle.open)        # gap-through: open below trigger fills at open
        fill = base - slip_price - spread_price
        # require candle.low  <= trigger to fill at all

# ---- STOP EXIT FILL (gap-through-stop = honest case) ----
LONG :  base = min(stop, candle.open)           # gap below stop fills at gap open
        fill = base - slip_price
SHORT:  base = max(stop, candle.open)           # gap above stop fills at gap open
        fill = base + slip_price

# ---- TARGET EXIT ---- (limit; fills at target, no positive slippage credited)
fill = target            # never better than target; pessimistic = no price improvement

# ---- COSTS recorded on the leg (D3), not folded into R ----
commission = commission_per_lot          # per round trip, 1 notional lot (A10 scales by size)
swap       = (swap_long_points if long else swap_short_points) * pip * nights_held

Note: the §10.3 4-bar expiry and the same-bar “stop-before-target” worst-case rule are unchanged from A6 — A7 only changes the price of a fill, never whether/when one happens, so look-ahead guards and expiry stay intact.

@dataclass
class PendingOrder:
    ...
    signal_ts: object
+   atr_state: str = "NORMAL"     # ATR state at signal time, for slippage scaling
+   atr_value: float = 0.0        # ATR (price units) at signal time

class SimulatedBroker:
    """§7.1 pessimistic fills: spread + ATR-scaled slippage + gap-through.
    EXTREME ATR state doubles slippage. Never fills better than the model."""

    def __init__(self, costs: BrokerCosts, pip_size: float):
        self.costs = costs
        self.pip = pip_size

    def _slip(self, order: PendingOrder) -> float:
        s = self.costs.slippage_atr_mult * order.atr_value
        return s * (2.0 if order.atr_state == "EXTREME" else 1.0)

    def try_fill(self, order, candle, bar_idx) -> Fill | None:
        spread = (candle.spread if candle.spread is not None
                  else self.costs.spread_points) * self.pip
        slip = self._slip(order)
        if order.direction == "long":
            if candle.high < order.trigger_price:
                return None
            base = max(order.trigger_price, candle.open)   # gap-through-trigger
            return Fill(base + slip + spread, bar_idx, candle.ts)
        else:
            if candle.low > order.trigger_price:
                return None
            base = min(order.trigger_price, candle.open)
            return Fill(base - slip - spread, bar_idx, candle.ts)

    def fill_stop(self, leg, candle, slip) -> float:
        """Honest stop-exit price incl. gap-through-stop (D1)."""
        if leg.direction == "long":
            return min(leg.stop, candle.open) - slip
        return max(leg.stop, candle.open) + slip

9 Open questions

  • Q1 (D3) ✓ Record-only. §7.1's cost mandate is satisfied by computing honest figures; the equity verdict (§7.2 G3 “profit factor OOS, after costs”) is an equity-level aggregation — A10's scope. Commission stored as currency/lot; swap as price-units (nights × swap_pts × pip) — documented on Position fields.
  • Q2 (D4) ✓ Full spread at entry only. Entry-only-full and half-entry/half-exit cost the same round-trip total; (a) is simpler. Exits already pay slippage + gap-through-stop — not flattered.
  • Q3 ✓ Fill exactly at target, no degrade. A limit fills at its price or better — degrading below target is wrong market mechanics. §7.1 names slippage only for entry stops and gap-through-stop; favourable gap-through-target still fills at target (decline the improvement, don't invent a worse fill).
  • Q4 ✓ Per-night calendar count for Phase A. §7.1 says “swap modelled per instrument” without specifying triple-Wed/rollover-time granularity. Swap is record-only (Q1), so the simplification is documented in the test and deferred to the B-track MT5 bridge.
