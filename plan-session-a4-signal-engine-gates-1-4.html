Session A4 — Signal Engine: Entry Gates 1–4

Implementation plan · 2026-06-17 · LSB Strategy Bot (Phase A, "Prove")

1 Overview

Implement the first four of the eight entry gates (module M7) as pure functions over candle windows — no broker, no side effects, no clock — built test-driven against hand-labeled fixtures. Gates 1–4 sit on top of supporting pure-function modules that must also be built here: M2 (EMA/ATR/candle classification), M3 (trend state), M4 (triangle structure), and M5 (liquidity blocks + sweep detection + 5-factor sweep score).

Every evaluation writes a signal row (JSONB gate_results) with per-gate pass/fail and a human-readable reason, keyed to the producing config_hash. Gates short-circuit: the first failure rejects the setup and records which gate failed and why. Gates 5–8 and the full eight-gate conjunction are deferred to Session A5.

Sources of truth: .scratch/LSB_System_Requirements_v2.0.md (immutable spec — §4 M2, §5 M3, §6 M4, §7 M5, §8 M7) and .scratch/blueprint_v2.1.txt §6.1. Gate grouping per LSB_Implementation_Plan_v3.0.md Part 1.

7 Failure modes

What could go wrongHow the plan handles it
EMA seeding ambiguity — recursive EMA needs a warm-up; different seeds → different values → fixture mismatch & nondeterminism.Fix the seeding convention explicitly (SMA seed over first period bars), document it, and require a minimum warm-up window before any gate evaluates. Reference-spreadsheet test pins it to 8 dp.
H4 resample drift — wrong session boundary or partial trailing bar shifts every H4 indicator.Pin H4 boundaries to fixed UTC hours; drop incomplete trailing H4 bar; reconciliation property test against source H1.
BTCUSD/synthetic pip semantics — "pips" vs "0.02%" differ for crypto/index vs FX; sweep penetration & block width use both.Drive penetration/width from pip_size + an asset_class branch (fx → pips, synthetic/crypto → percent). Spec §6.1/§7 says never share params across synthetic and conventional instruments.
Float accumulation order in scoring/indicators → nondeterministic hashes (the Gate GA-1 contract, A8).Pure functions, no dict-order-dependent sums, fixed iteration order; determinism is unit-asserted now so A8's CI gate inherits a clean base.
Scope creep — temptation to also wire gates 5–8 or backtest loop.Hard stop at gates 1–4 + their modules. Gates 5–8, conjunction, and shuffle property test are explicitly A5. Backtest is A6.
Look-ahead bias — a gate peeking at future candles inside the window.Engine receives a window ending at the evaluation candle; modules may only read up to that bar. Add a test that truncating future candles never changes a past evaluation.
