Implementation plan · 2026-06-20 · LSB / Phase A, session A1 · Issue #3

1 Overview

Build the LSB config system — YAML per-instrument files parsed into immutable frozen dataclasses, then hashed to a deterministic config_hash (sha256 of canonical JSON) that keys every downstream database row. Simultaneously author migrations/001_core.sql with the four Phase-A tables (config_version, candle, signal, wf_run + sim_trade), designed from first principles and recorded as ADR-002.

No gate logic, data fetch, or backtest code is introduced. The sole deliverable is a stable, deterministic config identity system that everything from A2 onward can anchor to.

3 Dataclass design (src/lsb/config/models.py)

InstrumentConfig — per-instrument, frozen=True, slots=True

@dataclass(frozen=True, slots=True)
class InstrumentConfig:
    instrument:        str        # "EURUSD" | "GBPUSD" | "XAUUSD" | "BTCUSD"
    instrument_class:  str        # "fx_major" | "commodity" | "crypto"
    pip_size:          Decimal    # stored as Decimal to avoid float repr ambiguity
    max_spread_pips:   Decimal    # in pip units for FX/gold; % for crypto stored as Decimal
    max_spread_unit:   str        # "pips" | "pct"
    sessions:          str        # "fx" | "24_7"
    flat_tolerance:    Decimal    # 0.15 (pct, spec §6.1.1 baseline)
    sweep_penetration: Decimal    # pips or pct depending on unit
    sweep_pen_unit:    str        # "pips" | "pct"
    block_min_width:   Decimal    # pips or pct
    block_width_unit:  str        # "pips" | "pct"
    stop_buffer:       Decimal    # normal ATR state
    stop_buffer_elev:  Decimal    # elevated ATR state
    stop_buffer_unit:  str        # "pips" | "pct"
    data_source:       str        # "dukascopy" | "binance"

⚠ Decimal is used throughout the hash path — never bare float. This guarantees no IEEE-754 repr drift between processes.

4 Hashing design (src/lsb/config/hashing.py)

Determinism invariant: same field values → same hash, regardless of Python version patch, process, or dict insertion order.

import hashlib, json
from decimal import Decimal
from dataclasses import asdict

def _canonical_value(v):
    if isinstance(v, Decimal):
        return str(v)          # "0.15", "0.0001" — stable, no float noise
    if v is None:
        return None
    return v                   # str, int — already stable

def config_hash(cfg) -> str:
    raw = {k: _canonical_value(v) for k, v in asdict(cfg).items()}
    canonical = json.dumps(raw, sort_keys=True, ensure_ascii=True)
    return hashlib.sha256(canonical.encode()).hexdigest()
RiskMitigation
float repr varianceAll numeric fields stored as Decimal; serialised via str()
dict orderingsort_keys=True in json.dumps
None handlingExplicit None → null in JSON; consistent across versions
Unicode driftensure_ascii=True forces consistent escaping

6 SQL schema (migrations/001_core.sql)

-- Phase-A core schema. Design rationale: ADR-002.

CREATE TABLE config_version (
    config_hash   TEXT        PRIMARY KEY,
    instrument    TEXT        NOT NULL,
    yaml_snapshot TEXT        NOT NULL,
    created_at    TIMESTAMPTZ NOT NULL DEFAULT now()
);

CREATE TABLE candle (
    id            BIGSERIAL   PRIMARY KEY,
    config_hash   TEXT        NOT NULL REFERENCES config_version(config_hash),
    instrument    TEXT        NOT NULL,
    timeframe     TEXT        NOT NULL,  -- 'H1', 'H4', 'D1'
    ts            TIMESTAMPTZ NOT NULL,
    open          NUMERIC     NOT NULL,
    high          NUMERIC     NOT NULL,
    low           NUMERIC     NOT NULL,
    close         NUMERIC     NOT NULL,
    volume        NUMERIC,
    UNIQUE (config_hash, instrument, timeframe, ts)
);

CREATE TABLE signal (
    id            BIGSERIAL   PRIMARY KEY,
    config_hash   TEXT        NOT NULL REFERENCES config_version(config_hash),
    instrument    TEXT        NOT NULL,
    ts            TIMESTAMPTZ NOT NULL,
    -- §8.1 gates (NULL = not evaluated; FALSE = fail; TRUE = pass)
    gate1         BOOL,  -- Trend State Confirmed
    gate2         BOOL,  -- Structure Confirmed
    gate3         BOOL,  -- Liquidity Sweep Detected
    gate4         BOOL,  -- EMA Interaction Confirmed
    gate5         BOOL,  -- Rejection Candle Confirmed
    gate6         BOOL,  -- Session Active
    gate7         BOOL,  -- Risk:Reward Minimum
    gate8         BOOL,  -- Global Risk State Clear
    all_gates     BOOL,  -- conjunction of gates 1-8
    sweep_score   NUMERIC,    -- 0-100 risk-tier score (not a gate; §7.3)
    risk_tier     TEXT,        -- '1.0pct' | '0.5pct' | '0.25pct' | 'skip'
    verdict       TEXT         -- 'QUALIFIED' | 'REJECTED' | 'INVALIDATED'
);

CREATE TABLE wf_run (
    id            BIGSERIAL   PRIMARY KEY,
    config_hash   TEXT        NOT NULL REFERENCES config_version(config_hash),
    instrument    TEXT        NOT NULL,
    train_start   DATE        NOT NULL,
    train_end     DATE        NOT NULL,
    test_start    DATE        NOT NULL,
    test_end      DATE        NOT NULL,
    CONSTRAINT no_overlap CHECK (train_end < test_start)
);

CREATE TABLE sim_trade (
    id            BIGSERIAL   PRIMARY KEY,
    wf_run_id     BIGINT      NOT NULL REFERENCES wf_run(id),
    signal_id     BIGINT      REFERENCES signal(id),
    entry_ts      TIMESTAMPTZ NOT NULL,
    exit_ts       TIMESTAMPTZ,
    entry_price   NUMERIC     NOT NULL,
    exit_price    NUMERIC,
    stop_price    NUMERIC     NOT NULL,
    target_price  NUMERIC     NOT NULL,
    r_multiple    NUMERIC,     -- positive = profit; negative = loss
    outcome       TEXT         -- 'WIN' | 'LOSS' | 'SCRATCH' | 'OPEN'
);

9 Failure modes

What could go wrongHow the plan handles it
Float repr drift breaks hash determinism across Python versions All numeric fields are Decimal; serialised with str() not repr(). test_float_stability verifies no 0.1000...1 drift.
XAUUSD pip_size / max_spread left as placeholders YAML file uses sentinel strings ("tbd"); loader raises ValueError on load until owner fills values. A2 data fetch cannot start for XAUUSD until resolved.
v2.0 Requirements doc later contradicts our Phase-A schema columns ADR-002 explicitly records the schema as a first-principles design. When the doc arrives, a 002_delta.sql migration is authored; no schema change is silent.
Owner-decision slots (min_trade_count etc.) accidentally hardcoded by a later session Fields are typed int | None / str | None; SpecConfig defaults to None. Any code reading them must handle None; verdict code that ignores the None check will raise at test time.
wf_run overlapping train/test windows CHECK (train_end < test_start) enforced at DB level.
