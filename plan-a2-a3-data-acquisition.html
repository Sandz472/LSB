A2–A3 — Data-acquisition module

Implementation plan · 2026-06-21 · LSB Phase A · Issue #7

1 Overview

Build the data-acquisition layer that feeds every downstream module: fetch_history (raw OHLCV from Dukascopy for EUR/GBP/XAU and Binance for BTC), audit_history (gap detection + mandatory disposition of every gap>2), a deterministic resample (H1 → H4/D1), and a load step that writes normalized candles into the candle table built in A1.

Per BUILD_PLAYBOOK §5 the accept-when is: EUR/GBP/XAU 3y Dukascopy + BTC 3y Binance, every gap>2 dispositioned, GBPUSD audit generated, counts reconcile. Correctness is proven by fixture-based unit tests before any bulk pull or live DB integration (CLAUDE.md §3); the multi-year download is an operational run, not a PR gate.

2 Approach & decisions

DecisionOptionsTrade-offChoice
Timeframe sourcing Native H1/H4/D1 pulls vs fetch H1 + resample Native = 3× source round-trips & reconciliation surface; resample = one canonical series, pure function Fetch H1, resample to H4/D1 — confirmed with owner; resampling is deterministic & unit-testable
Determinism of fetch Live fetch each run vs cache pinned raw downloads Live = non-reproducible audits; cache = byte-identical re-runs, offline tests Cache raw downloads to a pinned local store; audit/resample/load read the cache, never the network
Gap policy Drop/ignore gaps vs disposition every gap>2 Silent drop hides data quality; explicit disposition is auditable Every gap>2 classified (weekend/holiday/DST/genuine-missing); no silent gap → HALT if unclassifiable
Numeric type float vs Decimal/str float breaks byte-identical hashing; Decimal matches A1 precision path Decimal/str end-to-end, mirroring A1's loader
Load target coupling Direct psycopg writes vs thin loader + injectable connection Hard DB dependency blocks fixture tests; injectable sink enables round-trip test without a live DB Loader takes an injectable executor; round-trip tested against an in-memory/fake sink, real DB behind the CLI
Audit artifact format Free text vs structured (JSON/CSV) per instrument Structured = diffable, machine-checkable counts Structured per-instrument audit file (incl. required GBPUSD), with reconciling counts

4 CLI surface (no GUI — this is the operator view)

$ python -m lsb.data.cli fetch-audit-load --instrument GBPUSD --years 3

[fetch ] GBPUSD  dukascopy  H1   2023-06-21 → 2026-06-21   18,742 bars (cached)
[resamp] GBPUSD  H1→H4  4,690 bars   H1→D1  782 bars
[audit ] GBPUSD  gaps>2: 161  →  weekend 156 · holiday 4 · dst 1 · genuine 0
[audit ] wrote audit/GBPUSD_audit.json   counts reconcile ✓
[load  ] GBPUSD  H1 18,742 · H4 4,690 · D1 782   inserted (0 dup)
DONE  GBPUSD  source=18,742  audited=18,742  loaded=18,742  ✓

The three counts on the DONE line are the reconciliation invariant: source rows = audited rows = loaded rows (gaps are dispositioned, never dropped).

6 Failure modes

What could go wrongHow the plan handles it
Live network makes audits non-reproducibleRaw downloads cached to a pinned store; audit/resample/load read the cache only — tests run offline.
A gap>2 silently dropped → counts still "reconcile" falselyDisposition is total; an unclassifiable gap raises (no default bucket). Reconciliation compares source vs audited vs loaded, so a drop fails the invariant.
Crypto treated with FX weekend/holiday exemptionsclassify_gap branches on sessions == "24_7" (BTC) — no weekend/holiday excuse; matches §3.3 / CLAUDE.md.
float creeps into prices → hash divergence laterDecimal/str on the precision path end-to-end, asserted in load round-trip test.
DST boundary double-counts or skips an hourTimestamps normalized to UTC on fetch; DST is an explicit disposition class, not an implicit shift.
Re-running load duplicates rows (breaks A8 determinism)ON CONFLICT DO NOTHING on the table's UNIQUE key; idempotent re-run test asserts 0 duplicates.
Resample produces partial trailing bar (incomplete week/day)Resampler emits only complete buckets; partial-tail test pins the boundary behavior.
Scope creep into gates / broker pathsPackage is data-only; no signal/broker imports. Phase B locked in PHASE_STATUS.md.
