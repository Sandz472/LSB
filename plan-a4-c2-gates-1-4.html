A4 — C2 Signal Engine · Gates 1–4

Implementation plan · 2026-06-21 · LSB Phase A · issue #9 · branch feat/9-a4-c2-gates-1-4

1 Overview

Build the first four of the eight §8.1 gates that make up the C2 signal engine, implemented verbatim per GATE_SPECIFICATION.md (CLAUDE.md rule 1). Each gate is a pure, deterministic function over already-loaded OHLCV candles (Decimal, no wall-clock / randomness / dict-order — rule 4), evaluating the bearish short setup and its exact bullish mirror.

Scope is gates 1 Trend, 2 Structure (incl. the Invalidation rule), 3 Liquidity Sweep (vs the block HIGH), 4 EMA-Interaction. Gates 5–8 and the full conjunction are A5 — out of scope here. No DB writes, no backtest loop, no broker — gates are pure functions; the signal-row persistence boundary is A5 (Decision 3). Acceptance: every gate's pass-path is reachable on a hand-labelled fixture with ≥2 near-miss rejects, no silent dead gate, A4 establishes the golden gate-fixtures (the full-pipeline determinism replay is A8), and pytest -q is green (rules 5–6).

Swing-window half-width is per-instrument (liquidity-dependent), so it lives in InstrumentConfig.swing_lookback (= 2, ADR-005) — not in the universal StrategyParams. Added to all four YAMLs and the dataclass, since config_hash is computed over asdict(cfg) — a YAML-only key would never enter the hash.

src/lsb/signal/gate1_trend.py — D1 trend, all thresholds read from config (never typed)

def evaluate(
    candles_tf: Sequence[dict],      # candles on spec.trend_timeframe (D1, ADR-003)
    sp: StrategyParams,
    side: Side,
) -> GateResult:
    ema21, ema50, ema89 = ema(closes, 21), ema(closes, 50), ema(closes, 89)
    atr_v = atr(candles_tf, 14)       # D1 ATR(14) — same TF as the EMAs it scales (ADR-006)
    # INVALID blockers first (§5.2): compression OR recent EMA21/50 cross
    if abs(ema21 - ema89) < atr_v * sp.ema_compression_atr_mult:
        return GateResult(False, state="INVALID", detail={"reason": "compression"})
    if crossed_within(ema21, ema50, sp.ema_cross_lookback):
        return GateResult(False, state="INVALID", detail={"reason": "recent_cross"})
    # Stack + slope, mirrored by side (BEAR shown; BULL flips < to >)
    stack = ema21 < ema50 < ema89 if side is Side.BEAR else ema21 > ema50 > ema89
    slope_ok = slope_neg(ema21, sp) and slope_neg(ema50, sp)  # mirrored for BULL
    return GateResult(stack and slope_ok)

Gate 1 consumes spec.trend_timeframe candles (= "D1", ADR-003); Gate 4 consumes the §4.1 H1 EMAs. The two timeframes are read from config, never hard-coded.

6 Failure modes

What could go wrongHow the plan handles it
A threshold gets hard-coded in gate code (silent strategy drift — rule 1) All numbers live in StrategyParams/InstrumentConfig; a grep-for-numeric-literals test in tests/test_no_magic_numbers.py guards the decision path.
A gate's pass-path is unreachable — a "silent dead gate" Acceptance requires a green pass fixture and ≥2 near-miss rejects per gate; CI fails if any gate has no passing fixture.
float creeps into EMA/ATR → non-byte-identical decisions across machines (rule 4) Decimal end-to-end; a property test feeds the same candles twice and asserts identical GateResult.detail.
pip-vs-pct unit confusion (FX uses pips, crypto uses %) Unit travels with the value (*_unit fields already in InstrumentConfig); a single to_price_delta(value, unit, price, pip_size) helper, tested on FX + BTC.
Sweep accidentally targets a mean level, not the block HIGH (the ADR-006 prior-build drift) Gate 3 computes block_high = max swing-high wick explicitly; a regression test asserts the comparator is the block high, with the §R divergence cited in the test docstring.
D1 EMA89 needs ~89+ days of warm-up; early bars have no trend EMA returns None before the seed window; Gate 1 returns passed=False (not an exception). Fixtures include a warm-up-deficient case.
Bull mirror silently diverges from bear One parameterized function per gate; a mirrored-fixture test feeds a vertically-flipped bear fixture as BULL and asserts symmetric results.

7 Resolved decisions (owner, 2026-06-21)

  • ✓ Swing-window N = 2 (5-bar fractal: a pivot needs 2 strictly-bracketing bars each side). Per-instrument swing_lookback, hashed. Swept over N∈{1,2,3} at A9/A10 — a verdict that flips on N fails the §17.1 sensitivity check. → ADR-005
  • ✓ Gate 1 uses D1 ATR(14) for the slope (§4.1.2) and compression (§5.2) thresholds — same timeframe as the D1 EMAs it scales. H1 ATR is retained for all execution-scale uses (Gate 6 state, §7.3 sweep factor, §9.1 stop buffer, §9.4 R:R ATR×3) — do not switch those. → ADR-006 (extends ADR-003)
  • ✓ A4 gates are pure functions; no DB. A single persistence boundary at the A5 conjunction writes a signal row for 100% of evaluated candles (QUALIFIED/REJECTED/INVALIDATED) with per-gate reasons. No partial rows at A4. → BUILD_PLAYBOOK §5 (A4/A5 accept-when updated)
  • ✓ A4 establishes the golden gate-fixtures (hand-labelled candles → expected per-gate verdicts = correctness; frozen once owner-verified). The full-pipeline determinism replay is A8 / GA-1. No A0–A3 decision-golden exists (no gate logic before A4). → BUILD_PLAYBOOK §5 (A8 row clarified)
